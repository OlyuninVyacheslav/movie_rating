Хронология изменений исследовательского ноутбука по курсовому проекту
«Оценка рейтингов фильмов» (MovieLens 20M, PyTorch, регрессия).

## [2026-05-30] v0.1 — Создание структуры ноутбука

### Контекст
Ноутбук был пустым (0 байт). Цель — зафиксировать каркас (markdown + пустые
code-ячейки с комментариями-заглушками) перед написанием кода.

### Что было сделано
- Создан `movie_rating.ipynb` с **50 ячейками**: 20 markdown, 30 code.
- Определена архитектура эксперимента:
  - **Задача**: гибридная регрессия `user → movie`, целевая `rating ∈ [0.5, 5.0]`.
  - **Данные**: подвыборка ~1–2 млн строк (активные пользователи).
  - **Стек**: PyTorch 2.13 (CUDA), pandas, numpy, scikit-learn, matplotlib.

### Разделы ноутбука
| № | Раздел | Ячеек |
|---|--------|-------|
| 0 | Титул | 1 MD |
| 1 | Введение и постановка задачи | 4 MD |
| 2 | Импорт библиотек и конфигурация | 2 CODE |
| 3 | Загрузка данных | 1 MD + 2 CODE |
| 4 | Разведочный анализ (EDA) | 2 MD + 6 CODE |
| 5 | Предобработка данных | 1 MD + 4 CODE |
| 6 | Отбор значимых признаков | 1 MD + 3 CODE + 1 MD |
| 7 | Проектирование и реализация стенда | 1 MD + 4 CODE |
| 8 | Нейросетевые модели (3 шт.) | 1 MD + 3 CODE |
| 9 | Подбор гиперпараметров | 1 MD + 1 CODE |
| 10 | Обучение финальных моделей | 2 CODE |
| 11 | Оценка качества и сравнительный анализ | 1 MD + 3 CODE |
| 12 | Выбор лучшей модели и анализ результатов | 1 MD |
| 13 | Выводы | 1 MD |

### Модели (заложены как заглушки)
1. **Matrix Factorization** — dot-product эмбеддингов + смещения.
2. **NeuMF (Neural CF)** — конкатенация эмбеддингов → MLP.
3. **Hybrid** — эмбеддинги + отобранные контентные признаки → MLP.
+ **Baseline** (глобальное среднее + модель смещений) как точка отсчёта.

### Метрики (определены в разделе 1)
RMSE (основная), MAE, R².

## [2026-05-31] v0.9 — Раздел 8: нейросетевые модели

### Контекст
Заполнены 3 стуба раздела 8 (`cell-50`…`cell-52`). Все модели имеют единый
интерфейс `forward(users, movies)` (совместимы с `train_model`/`evaluate_model`
из раздела 7); гиперпараметры вынесены в `__init__` для подбора в разделе 9.

### Что было сделано
- **`cell-50` — `MatrixFactorization`.** Эмбеддинги user/movie + смещения b_u/b_i +
  глобальное среднее: `r = mu + b_u + b_i + <p_u, q_i>`. `global_mean` инициализирует
  старт около среднего по данным; малая инициализация эмбеддингов (std=0.05).
- **`cell-51` — `NeuMF`** + хелпер `make_mlp()`. Две ветви (GMF — поэлементное
  произведение; MLP — конкатенация эмбеддингов → персептрон), их выходы
  объединяются в скаляр. Реализует нелинейное взаимодействие.
- **`cell-52` — `HybridModel`.** Эмбеддинги user/movie + контентные признаки фильма
  (`content_tensor` как `register_buffer`, берётся по `movie_idx`) → MLP → оценка.
  Использует `make_mlp` из `cell-51`.

### Технические решения / нюансы
- Ноутбук превысил лимит чтения инструментом (>25k токенов), поэтому правка
  выполнена точечным Python-скриптом по JSON (поиск ячеек по маркерам-заглушкам,
  замена `source`, очистка `outputs`). JSON проверен на валидность (64 ячейки).
- `make_mlp` определён в `cell-51` и переиспользуется в `cell-52` — прогонять
  раздел сверху вниз.
- Контент в Hybrid — необучаемый буфер (перемещается на device вместе с моделью).
