## [2026-05-30] v0.1 — Создание структуры ноутбука

### Контекст
Ноутбук был пустым (0 байт). Цель — зафиксировать каркас (markdown + пустые
code-ячейки с комментариями-заглушками) перед написанием кода.

### Что было сделано
- Создан `movie_rating.ipynb` с **50 ячейками**: 20 markdown, 30 code.
- Определена архитектура эксперимента:
  - **Задача**: гибридная регрессия `user → movie`, целевая `rating ∈ [0.5, 5.0]`.
  - **Данные**: подвыборка ~1–2 млн строк (активные пользователи).
  - **Стек**: PyTorch 2.13 (CUDA), pandas, numpy, scikit-learn, matplotlib.

### Модели (заложены как заглушки)
1. **Matrix Factorization** — dot-product эмбеддингов + смещения.
2. **NeuMF (Neural CF)** — конкатенация эмбеддингов → MLP.
3. **Hybrid** — эмбеддинги + отобранные контентные признаки → MLP.
+ **Baseline** (глобальное среднее + модель смещений) как точка отсчёта.

### Метрики (определены в разделе 1)
RMSE (основная), MAE, R².

## Заметки по отчёту-эталону (отчёт по практике, диабет)

Изучен PDF `отчет_unlocked.pdf` — отчёт по учебной практике на тему
«Прогнозирование диабета» (задача ML, классификация). Ключевые наблюдения
для ориентира уровня курсовой:

- **Математическая постановка задачи** с формальными обозначениями (множества,
  функция отображения) — нужно воспроизвести для нашей регрессии в разделе 1.
- **Таблицы сравнения** моделей и методов (Таблица 1, 2) с колонками
  характеристик — аналог нужен в разделах 8 и 11.
- **Подбор гиперпараметров** через RandomizedSearchCV + StratifiedKFold с
  фиксацией времени выполнения и лучших параметров — у нас аналогично через
  ручной цикл или Optuna (PyTorch).
- **Три модели** (Logistic Regression, Random Forest, XGBoost) → у нас три
  нейросетевых архитектуры + baseline.
- **Интерпретация** через SHAP, Permutation Importance, LIME — у нас нужен
  аналог для нейросетей (Captum / SHAP для PyTorch, или просто анализ
  эмбеддингов + ошибок по сегментам).
- **Приложение с полным листингом кода** — учесть при оформлении отчёта.
- **Стиль**: формальный академический русскоязычный текст, без сокращений,
  ссылки на литературу в тексте ([1], [2]).
